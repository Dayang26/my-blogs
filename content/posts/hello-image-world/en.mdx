This blog post is a demo post illustrating how to reference and display static images in a blog system powered by Velite and Next.js.

## The Solution

When writing blog posts, we often need illustrations. If we reference static images using absolute root paths (e.g. `/static/...`):
1. **Local Development**: Next.js automatically maps the `public/` directory to the root `/`. Therefore, images placed in `public/static/` can be accessed locally directly via `/static/filename.png`.
2. **Cloudflare Pages Deployment**: After code is pushed to GitHub and automatically built by Cloudflare, Cloudflare distributes all static assets in the `public/` directory to its global CDN. When users request `/static/filename.png` on the webpage, it will point directly to the image on the CDN, realizing fast and stable loading.
