title: Pixel UI Design Guidelines
excerpt: Pixel UI is not retro for retro's sake—it's a re-balance of density, contrast, and structure.
---

## Principles

1. Pair large negative space with high-contrast blocks.
2. Keep copy short so the grid can breathe.
3. Use pixel shadows to communicate pressability.

## Modular Layout

- Hero tells the story, lists carry density.
- Featured cards use heavier borders.
- Newsletter stands alone to avoid being ignored.

## Conclusion

Pixel UI works when order is visible and intentional.
