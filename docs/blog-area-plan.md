# Blog 区域构建计划（像素风）

## 目标

- 点击首页“日志区域”按钮进入 Blog 区域
- Blog 区域使用像素风视觉，但布局/信息架构符合现代博客标准
- 先产出结构与静态内容，便于后续接入真实数据

## 方案概览

- 路由：使用 `/blog` 作为主入口，同时让 `/logs` 指向 Blog 区域以保持入口一致
- 页面结构：Hero + Featured + 分类/标签 + 文章列表 + 订阅区 + Footer
- 数据：本地静态数组（标题、摘要、标签、日期、阅读时长）
- 视觉：像素边框、块状卡片、粗颗粒阴影、像素按钮

## 子任务与步骤

### 1. 路由与入口调整

1.1 新建 `app/blog/page.tsx`

1.2 更新首页日志入口按钮，指向 `/blog`

1.3 将 `app/logs/page.tsx` 调整为 Blog 页面或转向 `/blog`

验收：
- 点击“日志区域”进入 Blog 页面
- 直接访问 `/logs` 也能看到 Blog

### 2. Blog 页面结构与内容

2.1 Hero 区：标题、说明、主按钮

2.2 Featured 区：突出 1-2 篇内容

2.3 标签/分类区：像素按钮样式

2.4 文章列表：卡片式布局（标题、摘要、标签、日期、时长）

2.5 订阅区：邮件输入 + 按钮（静态）

验收：
- 内容结构完整
- 文案清晰、可替换

### 3. 像素风视觉与组件样式

3.1 使用 `pixelated` 与像素边框/投影

3.2 标题字体/间距更具“游戏UI”感

3.3 卡片/按钮统一外观

验收：
- 整体风格统一
- 与首页像素风一致但更“博客”

### 4. 质量检查

4.1 `pnpm lint`

4.2 页面访问 `/blog` 和 `/logs`

验收：
- lint 通过
- 页面展示正确
