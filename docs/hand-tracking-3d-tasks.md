# 手势追踪 3D 升级 - 任务指导书

> 目标：对现有实现进行文档同步、手势稳定性增强、UI 产品化和发布流程收尾。

## 任务 1：同步/修正文档与现有实现

**目标**：所有文档与当前代码结构、功能、依赖保持一致。

**子任务**：
- 更新 README：项目定位、主要入口路由、启动方式
- 更新 `docs/desc.md`：实际功能概述、目录结构
- 更新 `docs/hand-tracking-3d-architecture.md`：组件与数据流、文件路径
- 删除或替换过期描述

**实施步骤**：
1. 盘点当前目录结构与功能模块
2. 修正文档中的路径/组件名称
3. 补充手势稳定性、手势判定逻辑等最新行为

**验收标准**：
- 文档中不再出现不存在的文件/目录
- 描述与实际代码一致
- 阅读文档即可准确了解当前功能

---

## 任务 2：手势逻辑稳定性增强

**目标**：支持左右手差异判断、平滑滤波、置信度阈值控制。

**子任务**：
- 引入左右手判定，优化拇指伸展判断
- 增加 landmark 平滑（指数移动平均）
- 引入手部检测置信度阈值
- 引入手势置信度阈值

**实施步骤**：
1. 扩展 `HandTrackingConfig` 配置项
2. 在 `useHandTracking` 中增加置信度过滤与平滑处理
3. 在 `gesture-detector` 中加入 handedness 逻辑
4. 校验交互逻辑是否受影响

**验收标准**：
- 左/右手在捏合、握拳时判定合理
- 抖动显著降低
- 低置信度时不触发抓取

---

## 任务 3：产品化 UI 与路由入口

**目标**：提升 `/hand-3d` 页面呈现和首页入口体验，使其更像产品 demo。

**子任务**：
- 首页提供清晰入口和功能说明
- `/hand-3d` 增加状态信息与控制面板
- 响应式布局与视觉一致性

**实施步骤**：
1. 更新 `app/page.tsx`，增加 demo 入口
2. 优化 `app/hand-3d/page.tsx` 的布局与控制栏
3. 调整应用 metadata 与基础样式

**验收标准**：
- 首页能一键进入 hand-3d
- 关键状态（摄像头/检测/手势/FPS）可见
- 页面在桌面与移动端都可用

---

## 任务 4：合并与发布

**目标**：将当前分支合并到 `main` 并推送，触发 Cloudflare Pages 自动部署。

**子任务**：
- 提交当前分支修改
- 合并到 main，解决冲突（如有）
- 推送到 GitHub

**实施步骤**：
1. `git add` 与 `git commit`
2. `git checkout main` 并 `git merge` 当前分支
3. `git push origin main`

**验收标准**：
- `main` 分支包含全部修改
- 远端成功更新
- Cloudflare Pages 能正常构建

---

## 推荐执行顺序

1. 任务 2：手势逻辑稳定
2. 任务 3：UI 与入口
3. 任务 1：文档同步（最后统一更新更准确）
4. 任务 4：合并与发布
