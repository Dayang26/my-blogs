# 像素风主页改造 - 子任务分解与步骤

## 任务目标

将主页改造成像素游戏风格，并提供两个入口按钮：
- 进入试验区（/hand-3d）
- 日志区域（/logs）

## 子任务 1：资源与样式基座

**目标**：让页面具备像素风视觉基础与图片渲染效果。

**步骤**：
1. 在全局样式中加入 `pixelated` 图片渲染类
2. 定义可复用的像素边框与按钮样式（使用 Tailwind 组合类）
3. 确认 `public/img.png`、`public/rw.png`、`public/fire.png` 可以直接访问

**验收标准**：
- 页面图片呈像素化渲染
- 视觉风格可被复用到主页与日志页

## 子任务 2：主页布局与视觉重写

**目标**：完成像素游戏风格的主视觉与布局。

**步骤**：
1. 将 `app/page.tsx` 替换为新的像素风布局
2. 使用 `img.png` 作为全屏背景
3. 将 `rw.png` 作为主角精灵放置在左下区域
4. 将 `fire.png` 作为场景元素放置在角色附近
5. 增加标题与简短说明

**验收标准**：
- 主页整体风格为像素游戏
- 背景、人物、火焰元素位置合理
- 页面在桌面与移动端均可阅读

## 子任务 3：入口按钮与路由

**目标**：提供清晰入口，引导进入试验区与日志区。

**步骤**：
1. 在主页右侧放置两个按钮
2. 按钮 1 跳转到 `/hand-3d`
3. 按钮 2 跳转到 `/logs`
4. 按钮采用像素风视觉（边框、投影、按压反馈）

**验收标准**：
- 按钮可点击并正确跳转
- 按钮视觉与主页一致

## 子任务 4：日志区域页面

**目标**：新增 `/logs` 页面，保持像素风。

**步骤**：
1. 创建 `app/logs/page.tsx`
2. 使用像素风背景与文字
3. 提供返回主页入口

**验收标准**：
- `/logs` 页面可访问
- 与主页视觉一致
- 有明显返回主页入口

## 子任务 5：质量检查

**目标**：确保编译与 lint 不报错。

**步骤**：
1. 运行 `pnpm lint`
2. 运行 `pnpm run build`

**验收标准**：
- lint 通过
- build 通过
