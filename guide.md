# 博客写作指南 (Blog Writing Guide)

在本项目中，博客内容的管理基于 [Velite](https://velite.js.org/)。系统会自动从 `content/posts/` 目录下抓取特定格式的文件进行渲染。

## 如何新建一篇博客？

你只需要按照以下 3 个简单的步骤即可发布或预览新文章：

### 第一步：复制模板文件夹
项目内已经为你准备好了标准模板。进入 `content/posts/` 目录，将 `_template` 文件夹**复制一份**。
将复制后的新文件夹重命名为你这篇新文章的英文缩写（这会作为 URL 的路径后缀，也就是 Slug）。
例如：你想写一篇关于 React 状态管理的文章，可以命名为 `content/posts/react-state-management`。

### 第二步：配置文章元数据 (`meta.json`)
进入你刚刚创建的文件夹，打开 `meta.json`。这个文件用来控制文章的发布状态和分类等配置信息。
```json
{
  "date": "2026-05-26",     // 发布日期
  "tags": ["React", "前端"], // 标签列表
  "featured": false,        // 是否标记为精选文章
  "type": "DeepDive",       // 分类：例如 DeepDive, Essay 等
  "status": "published"     // 发布状态
}
```
> **提示**：如果文章还没写完，不想在正式环境展示，可以将 `"status"` 改为 `"draft"`（草稿）。

### 第三步：编写正文 (`zh.mdx` / `en.mdx`)
打开 `zh.mdx`（中文版）或 `en.mdx`（英文版）。
在文件的最顶部有一块被 `---` 包裹的 YAML 区域（Frontmatter），在这里填入你的文章标题和简短摘要。
```mdx
---
title: "我的第一篇博客"
excerpt: "这篇博客的简短摘要，将会展示在博客的首页列表和卡片中..."
---
```

接着在 `---` 区域下方，就可以正常编写你的 Markdown 文本了。
由于项目支持 MDX，所以不仅支持标准 Markdown 语法，还可以直接渲染 React 组件（例如内置的 `<Video />` 组件、插入代码高亮块等）。

### 预览你的成果
确保你在终端中已经启动了本地开发服务器：
```bash
pnpm dev
```
在你敲字、修改或者保存这些文件时，Velite 引擎会自动触发热更新。你只需切换回浏览器，就能实时预览你新鲜出炉的文章啦！
